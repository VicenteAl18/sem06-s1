Desventajas de CCS plano

CSS plano genera códigos excesivos y muchas veces difíciles de comprender.

¿Qué es un pre - procesador?

Es un programa que te permite generar CSS a partir de la syntax única del preprocesador, la mayoría de preprocesadores CSS añadiran algunas características que no existen en CSS puro como anidamiento, nesting, modularidad, entre otros. Estas características hacen la estructura de CSS más legible y fácil de mantener.

¿Qué es SASS?

Sass es un preprocesador CSS es una herramienta que nos permite permite tener una organización modular de los estilos, lo cual es vital para proyectos grandes y reduce el tiempo para crear y mantener el CSS.

♦ Compilación:

Para la compilación de SASS primero se debe instalar la extensión Live Sass Compiler , seguido de ello crear un archivo.scss. en la parte inferior de OUTPUT se habilitara la opción WATCH SASS, al dar click ahí automateicamente generará 2 archivos más, un .css y .css.map

♦ Variables:

Para un desarrollo ordenado y rápido en el archivo .sccs se crea variables con el simbolo ($), con estas se puede crear varibales de colores, tamaños, efectos, entre otros Por ejemplo: En esta vista tenemos variables de colores como: $first-color: red; $secund-color: yellow; $third-color: white. Tambien de tamaños $xxl:100px; $xl:50px.

♦ Anidamiento:

Para dar a estilos a varias etiquetas de una sección dento de la etiqueta su pone + y otra etiqueta con los elementos que se desea dar al CSS. Ejemplo: Aquí estamos anidando
h3 {
color: $first-color;
font-size: 30px;
padding-left: 50px;
+ p{
color: $third-color;
span{
color: $fourth-color;
}
}
}

♦ Nesting:

Cuando nombramos clases para dar estilos a la relación padre e hijo de las clases normalmente se hace .padre-hijo. En SASS para hace que el anidamiento sea corto se usa el simbolo (&) quedando &-hijo.
